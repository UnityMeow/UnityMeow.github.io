结构体和类的区别是什么

类型、成员声明、变量初始值、构造析构

  • 结构体是值类型，类是引用类型
  • 结构体成员不可被声明为protected，类成员可以
  • 结构体变量声明不能指定初始值，类变量声明可以指定初始值
  • 结构体不能声明析构以及默认的构造函数，类无此限制

值类型和引用类型的区别是什么

继承类型、内存分配、存取速度、存储内容、释放机制、默认赋值、赋值内容

  • 值类型隐式继承自System.ValueType而且不能继承其他任何类型，引用类型继承自System.Object
  • 值类型在栈中分配内存，引用类型在堆中分配内存，引用中存放的是堆中存放的地址
  • 值类型存取快，引用类型比值类型存取慢
  • 值类型表示实际数据，引用类型表示指向存储在对内存中的数据的指针
  • 栈内存是自动释放的，堆内存是由.Net中的GC来自动释放的
  • 值类型不能为null，必须有一个确定的值，引用类型在被赋值前都为null
  • 赋值值类型变量是赋值变量的值，引用类型相当于是赋值变量的地址

ref参数和out参数的作用是什么以及有什么区别

  • ref 必须先赋值 out 不需要

ArrayList和List的主要区别

  • ArrayList是普通类，会隐式转换成object，存在装箱拆箱的可能，存在类型不安全。
  • List是泛型类，在使用时必须填装入类型，编译器对于泛型会预编译额外代码避免拆装箱，处理效率高于ArrayList

请简述关键字sealed的作用

  • 类声明时使用sealed可以防止其它类继承此类
  • 方法声明时使用，可以防止子类重写此方法，且必须要与override一起使用

Heap和Stack有何区别

  • Heap是堆，Stack是栈
  • Heap的空间是手动申请和释放，用关键字new来分配，Stack的空间由操作系统自动分配和释放
  • Heap空间大，相对自由，Stack的空间有限

GC产生的原因以及如何避免

  • 如果有堆内存被系统判定为不再使用，那么由垃圾回收器GC进行回收
  • 避免产生堆内存对象，减少new产生对象的次数
  • string可改为StringBuilder

什么是委托，有什么作用

  • 委托也是一种数据类型，是存的方法的引用

  • 委托变量可以承载一个或多个方法

  • 委托( Delegate )是存有对某个方法或多个方法引用的一种引用类型

  • 所有的委托( Delegate )都派生自System.Delegate类

  • 委托本质上就是方法的列表（有顺序）

  • 事件中心，观察者模式

  • 委托类型可以作为 一个参数传给一个方法

  • 委托可以将方法当做一个变量去使用

  • 一个委托变量，在另一个方法中去调用叫做回调

委托与事件的区别是什么

  • 事件相当于一个封装好的委托，使用更安全

什么是反射，反射的实现原理是什么*

  • 可以在运行的时候得到类内的函数和属性

请简述private，public，protected，internal的区别

  • private：私有 成员只对其所在类公开
  • protected：保护 成员只对其所在类以及子类公开
  • public：公共 对任何类和成员都公开，没有访问限制
  • internal：内部 只能在包含该类的程序集中访问

请描述Interface与abstarct类之间的不同

  • 接口不能有构造函数，抽象类可以有构造函数，提供给子类使用
  • 接口可以实现多继承，抽象类只可单继承
  • 接口只能有静态的不能被修改的数据成员，一般不定义数据成员，抽象类可以有自己的数据成员
  • 接口所有的成员方法都是abstract的，抽象类可以有非abstarct的成员方法
  • 继承接口的类必须实现接口的所有方法，继承抽象类的类必须实现所有的abstract方法

简述StringBuilder和String的区别？

  • String中的方法每次都需要穿件一个新的字符串对象，并且分配新的内存地址，StringBuilder是在原来的内存里对字符串进行修改
  • String主要用于公共API，通用性好、用途广泛、读取性能高、占用内存小，StringBuilder主要用于字符串拼接，修改性能好
  • String是不变的，线程同步，StringBuilder可变，非线程同步

什么是装箱拆箱，如何优化他们的效率

  • 装箱：值类型转为引用类型的过程叫装箱

    装箱流程*

    1. 在堆内存中分配值类型各字段加类型对象指针，同步块索引所需的内存空间
    2. 值类型的所有字段复制到新分配堆内存中
    3. 返回该对象的地址，也就是说该对象的引用

  • 拆箱：引用类型转为值类型的过程叫拆箱，但如果变量为null或引用类型指向的不是期待值类型的已装箱实例，会抛出异常

  • 尽可能减少值类型与引用类型之间的转换与隐转

什么是泛型，泛型的约束有什么用？

  • 泛型允许我们延迟编写类或方法中的编程元素的数据类型的规范，直到实际在程序中使用它的时候。也就是说，泛型是可以与任何数据类型一起工作的类或方法。
  • 约束告知编译器类型参数必须具备的功能

什么是协变逆变？

  • 要把泛型类型参数定义为协变，可以在类型定义中使用out关键字，协变类型参数只能用作方法的返回值或属性get访问器
  • 要把泛型类型参数定义为逆变，可在类型定义中使用in关键字，对于接口定义，抗变类型参数只能用作方法参数，不能用作返回类型

同步与异步的区别是什么？

  • 同步：程序运行的时候，在调用其他方法的时候，会等待被调用的方法按顺序执行完，才会继续执行。
  • 异步：在程序调用异步方法的时候，主程序不会等待方法执行完，而是主程序调用异步方法后直接继续运行，而异步方法会启动一个新线程来完成方法的计算。

什么是线程？什么进程？

  • 一个进程会包含多个线程：线程是隶属于某个进程，进程销毁，线程则消失

多线程的优缺点是什么？

  • 优点：

    可以提高CPU的利用率。在多线程程序中，一个线程必须等待的时候，CPU可以运行其它的线程而不是等待，这样就大大提高了程序的效率。

  • 缺点

    1. 线程也是程序，所以线程需要占用内存，线程越多占用内存也越多；
    2. 多线程需要协调和管理，所以需要CPU时间跟踪线程；
    3. 线程之间对共享资源的访问会相互影响，必须解决竞用共享资源的问题；
    4. 线程太多会导致控制太复杂，最终可能造成很多Bug；
